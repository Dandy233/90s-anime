# Heartbeat 检查清单 v2.0

## 日常检查（2-4次/天）
- [ ] **系统状态** - gateway运行是否正常 (`openclaw gateway status`)
- [ ] **cron任务** - 检查是否有失败的定时任务 (`openclaw cron list`)
- [ ] **磁盘空间** - 日志和会话文件是否占用过多 (`df -h`)
- [ ] **git状态** - workspace是否有未提交的改动 (`git status`)

## 信息监控（按需）
- [ ] **邮件/消息** - 是否有需要处理的紧急信息
- [ ] **系统日志** - 过去几小时是否有错误或警告 (`openclaw logs`)
- [ ] **会话健康** - 会话文件是否过大需要清理

## 周期性维护（每周）
- [ ] 整理 memory/ 目录，归档旧日志
- [ ] 更新 MEMORY.md，提炼有价值的信息
- [ ] 检查 workspace 是否有需要清理的文件
- [ ] 审查定时任务运行历史

## 智能触发条件
- 时间：避开深夜(23:00-08:00)除非紧急情况
- 频率：同一检查间隔不少于30分钟
- 静默：无新内容时回复 HEARTBEAT_OK

## 自动化状态
当前已配置4个定时任务自动执行：
- ✅ daily-it-news: 信息获取
- ✅ weekly-system-check: 系统监控
- ✅ auto-git-sync: 代码备份
- ⏳ delayed-surprise: 知识更新
