# 子代理任务分发系统

## 概念
将复杂任务分解为多个子任务，由独立的子代理并行处理，最后汇总结果。

## 使用场景

### 场景1: 多源信息收集
- 子代理A: 搜索技术文档
- 子代理B: 搜索社区讨论
- 子代理C: 搜索新闻动态
- 汇总: 整合所有信息生成报告

### 场景2: 多维度分析
- 子代理A: 分析技术可行性
- 子代理B: 分析商业可行性
- 子代理C: 分析风险评估
- 汇总: 综合决策建议

### 场景3: 批量处理
- 子代理1-N: 并行处理多个文件/任务
- 汇总: 收集所有结果

## 实现方式

### 方法1: Cron + Isolated Session
```json
{
  "name": "parallel-research",
  "schedule": { "kind": "at", "at": "2026-03-01T10:00:00Z" },
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "研究任务A: ..."
  }
}
```

### 方法2: 主代理协调
主代理依次调用 sessions_spawn 启动多个子代理，等待结果后汇总。

## 最佳实践

1. **任务独立性**: 每个子代理任务应该独立，不依赖其他子代理
2. **超时设置**: 为子代理设置合理的超时时间
3. **错误处理**: 考虑部分子代理失败的情况
4. **结果汇总**: 设计清晰的汇总逻辑
